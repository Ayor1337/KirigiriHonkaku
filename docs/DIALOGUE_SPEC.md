# 对话模型规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/GAME_SPEC.md) 中拆出对话相关设计。
> 当前只记录对话系统的大框架，不展开字段级细节。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档当前只回答以下问题：

- 对话在这个游戏里承担什么作用
- 对话为什么必须双轨保存
- 对话和 NPC、时间、暴露度之间是什么关系

## 2. 总体结论

当前对话系统可以概括为：

**对话是玩家推进案件的核心行动之一，它既是获取信息的方式，也是暴露自身、改变局势、影响 NPC 记忆与后续行为的重要入口。**

## 3. 对话的定位

当前约定：

- 玩家主要通过和多个 NPC 分别对话来推进调查
- 对话不是纯展示文本，而是会真正改变世界状态的行动
- 对话既可能带来信息，也可能带来风险

这意味着：

- 对话是主循环的一部分
- 对话不是附属 UI，而是核心调查手段
- 玩家是否、何时、如何与谁对话，会直接影响案件推进

## 4. 对话与时间

当前约定：

- 对话会消耗时间
- 对话必须和时间绑定
- 对话应能区分所属场景或调查阶段

这意味着：

- 对话不是脱离时间轴的静态节点
- 同一个 NPC 在不同时间点的对话价值可能完全不同
- 某次对话发生的时机，本身就是案件结构的一部分

## 5. 对话与 NPC

对话和 NPC 的关系不是一次性的问答关系，而是持续影响。

当前理解中：

- 对话会影响 NPC 的动态记忆
- 对话会改变 NPC 对玩家的判断
- 对话可能改变 NPC 后续的配合度、戒备心或行为倾向

这意味着：

- 每次对话都不只是“取信息”
- 对话也在向 NPC 暴露玩家的兴趣、立场和调查方向

## 6. 对话与风险

当前约定：

- 对话可能提高暴露度
- 反复逼问同一 NPC、频繁打听敏感问题、在多人面前公开怀疑某人，都会让对话从信息获取手段变成风险来源

这意味着：

- 对话不仅是信息收益行为
- 也是风险管理行为
- 玩家不能把“问更多问题”视为零成本操作

## 7. 对话的双轨保存

当前约定：

- 对话必须同时保存摘要和完整记录

原因如下：

- 摘要用于后续 Agent 上下文压缩，控制 token 成本
- 完整记录用于前端展示、回放、调试与追踪

因此对话保存不是“冗余设计”，而是功能分工：

- 摘要负责延续系统理解
- 全文负责保留完整交互现场

## 8. 对话与玩家认知

对话不仅是玩家获取线索的方式，也是玩家认知层的重要来源之一。

当前理解中：

- 对话可以带来新信息
- 对话也可以让玩家重新解释旧信息
- 对话中的证词、态度、回避、失态，本身都可能成为线索

这意味着：

- 对话不能只被理解成“拿台词”
- 它本身也能产出线索、异常和怀疑对象

## 9. 对话与公开场合

当前约定：

- 对话既可以发生在私下
- 也可以发生在公开场合

而公开场合中的对话会天然带来额外效果，例如：

- 形成旁观者
- 影响群体判断
- 改变后续局势
- 放大成功或失败的影响

这意味着：

- 对话不只是“玩家和一个 NPC 的聊天”
- 它有时是社会场景的一部分

## 10. 当前结论

对话系统当前已经可以先按下面这句话理解：

这是一个以“时间绑定 + 风险绑定 + NPC 记忆更新 + 摘要 / 全文双轨保存”为核心的大框架，对话既承担信息获取功能，也承担暴露、试探、施压和改变局势的功能。

后续如果继续细化，优先讨论：

1. 对话的类型如何划分
2. 对话对 NPC 记忆和态度的影响如何表达
3. 对话与线索、公开场合、正式指认之间的衔接方式
