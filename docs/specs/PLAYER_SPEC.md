# 玩家模型规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/specs/GAME_SPEC.md) 中拆出玩家相关设计。
> 当前目标是先定义玩家模型的骨架、首批模板方向与信息边界，不在这一版中细抠数值平衡。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档回答以下问题：

- 玩家作为游戏主体，需要被建模成哪些层次
- 玩家预设身份应如何影响调查体验
- 玩家属性、资源、特质与知识应如何分层
- 首批可选模板应该长什么样

本文档当前是玩法与数据设计文档，不是实现说明。

## 2. 设计原则

- 玩家模板必须有明显路线差异，但不应变成完全不同剧本
- 属性系统保持克制，不做过多派生能力展示
- AI 需要在玩家塑造上保有足够戏份，因此文本描述优先于细碎规则
- 玩家模型既要支持结构化状态，也要支持叙事化表达
- 先做 4 个强辨识度模板，不急于扩充职业数量

## 3. 玩家模型总览

玩家模型当前按四个逻辑域理解：

- 稳定层
- 动态层
- 认知层
- 工具层

这四层是逻辑边界，不强制等于四个文件或四张表。后续实现允许继续细拆。

### 3.1 稳定层

稳定层用于定义“这个玩家是谁”。

至少包含：

- 身份背景
- 基础属性
- 初始特质
- 初始资源
- 初始关系

### 3.2 动态层

动态层用于定义“玩家现在处于什么状态”。

至少包含：

- 当前生命 / 伤势 / 中毒等生存状态
- 当前时间相关状态
- 当前所在地点
- 当前暴露度
- 当前持有物品与资源变化
- 当前临时效果
- 当前已解锁区域或可通行权限

### 3.3 认知层

认知层当前不再强行拆分“已知事实”和“已获得线索”，而是先合并成统一知识池。

这个统一知识池的组织方式为：

- 底层按来源记录
- 上层允许按主题归类

当前至少覆盖：

- 玩家已经接触并纳入认知范围的信息
- 玩家做过的关键行动
- 玩家接触过的 NPC、地点、事件信息
- 玩家听过的重要证词摘要

### 3.4 工具层

工具层用于承接玩家的整理行为，而不是世界事实本身。

至少包含：

- 自由笔记
- 侦探板卡片
- 侦探板连线
- 侦探板分组 / 布局
- 侦探板自由文本
- 玩家自定义标签

## 4. 属性系统

### 4.1 底层属性

玩家底层属性采用 DND 风格六维：

- 力量
- 敏捷
- 体质
- 智力
- 感知
- 魅力

对玩家展示时，只展示这六维属性。

### 4.2 属性区间

六维属性当前倾向采用接近 DND 的区间，例如 `3-18`。

当前不采用：

- 过于扁平的 `1-5`
- 过于精细的百分制

### 4.3 内部判定能力

虽然前端只展示六维属性，但系统内部仍可用更少量的核心能力来承接判定。

当前先收敛为四类：

- `调查`：偏 `智力 + 感知`
- `交涉`：偏 `魅力`
- `行动`：偏 `敏捷 + 体质`
- `压制`：偏 `力量`

这些能力当前只作为内部理解层，不需要额外展示给玩家。

## 5. 特质系统

### 5.1 当前定位

玩家需要“特质”这一层，但当前不把特质做成重规则系统。

特质目前的定位是：

- 文本描述
- 人物气质刻画
- AI 对玩家身份与风格的参考
- 对话和叙事风格的辅助约束

当前不把特质作为：

- 正式数值修正器
- 明确的规则开关
- 复杂 build 系统的一部分

### 5.2 来源

特质采用混合式来源：

- 身份模板自带少量核心特质
- 玩家开局再补选少量个人特质

但这些特质当前主要服务于文本与角色扮演，不承担强机制职责。

## 6. 初始资源系统

玩家初始资源至少分为以下五类：

- 随身物品
- 武器 / 自保工具
- 身份资源
- 金钱 / 可交易资源
- 特殊资料

### 6.1 模板固定给予

以下资源更适合由模板固定给予：

- 身份资源
- 特殊资料

### 6.2 开局可微调

以下资源允许玩家在开局做轻度微调：

- 随身物品
- 武器 / 自保工具
- 少量金钱分配

## 7. 身份模板机制

### 7.1 模板数量

首批玩家模板控制在 `3-4` 个。

### 7.2 模板差异方式

身份模板差异采用混合式，不只靠属性区别。

每个模板应同时体现：

- 属性倾向
- 初始资源
- 初始关系
- 初始权限
- 初始已知信息
- 少量专属行动或对话入口

### 7.3 首批模板风格

首批模板先采用偏经典推理风格，而不是偏奇幻冒险职业池。

### 7.4 模板差异强度

首批模板采用“强差异”设计：

- 会明显影响可见信息
- 会明显影响可走路线
- 会明显影响可做动作
- 但不会强到变成完全不同主线

### 7.5 自由分配力度

身份模板采用“预设模板 + 少量自由分配”。

当前约束如下：

- 自由分配同时影响属性和初始资源
- 但调整力度应很轻
- 模板主体不能被改得面目全非

## 8. 首批四个模板

首批模板先按四个功能位设计，再用背景卡名承载。

### 8.1 真相追猎者

定位：

- 默认推荐模板
- 标准调查型
- 偏纯调查，不是全能模板

核心体验：

- 更擅长发现问题
- 更容易抓到细节
- 更容易沿着线索追问
- 在强压、强闯和正面风险应对上不占优势

设计方向：

- 高项优先落在 `智力`、`感知`
- 资源偏向笔记、资料、调查工具
- 适合作为新玩家理解核心玩法的基准模板

### 8.2 秩序执行者

定位：

- 官方权限型
- 拥有更强的正式身份与秩序资源

核心体验：

- 更容易接触正式信息
- 更容易施加制度压力
- 更容易进入部分受限区域
- 但行动可能更显眼，也更容易被视为强势介入者

设计方向：

- 属性倾向偏 `魅力`、`体质` 或较稳定的综合型
- 资源偏向证件、通行权限、制度性背书
- 在公开场合与正式接触上天然更有优势

### 8.3 人群中的耳目

定位：

- 社会渗透型
- 擅长从关系、传闻、舆论与话术中获取信息

核心体验：

- 更擅长套话
- 更擅长在多人场合影响气氛
- 更容易从边缘人物和零碎信息中得到线索
- 但在物证调查和高风险场景中不如调查型和官方型稳定

设计方向：

- 属性倾向偏 `魅力`、`感知`
- 资源偏向人脉、消息渠道、轻便随身物
- 在伪证、舆论、群体场合中的潜力更高

### 8.4 异状解读者

定位：

- 专业知识型
- 擅长把“别人看不懂的异常”转化成有效信息

核心体验：

- 更容易理解尸体、药物、痕迹、文书、现场细节
- 更容易识别专业层面的异常
- 在部分场景中可比其他模板更早读出关键问题
- 但在社会推动力和强势介入上不一定占优

设计方向：

- 属性倾向偏 `智力`，其次看具体子职业落在 `感知` 或 `魅力`
- 资源偏向专业器具、记录材料、说明类资料
- 适合作为“从专业视角破案”的路线模板

## 9. 当前不急于拍板的内容

以下内容当前不急于细化：

- 四个模板的精确数值分布
- 各模板可选的具体个人特质清单
- 各模板可微调的具体点数与资源额度
- 玩家开局 UI 的具体呈现方式
- 玩家认知层最终的表结构与索引结构

## 10. 当前结论

玩家模型当前已经可以先按下面这句话理解：

这是一个以“四层玩家状态 + 六维属性 + 轻规则特质 + 五类资源 + 四个强差异身份模板”为核心骨架的玩家设计方案，其中 AI 主要负责放大身份气质、对话风格与叙事路线差异。
