# 地图模型规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/specs/GAME_SPEC.md) 中拆出地图相关设计。
> 当前只记录地图系统的大框架，不展开字段级细节。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档当前只回答三个问题：

- 地图分层保留到什么程度
- 地图开放逻辑采用什么节奏
- 聚餐、集会、公开指认这类场合如何挂在地图上

## 2. 总体结论

当前地图系统可以概括为：

**一个允许混合分层的图模型世界，主路径大多可达，关键区域逐步开放，而聚餐、集会、公开指认等场合则作为依附地点的事件层来运作。**

## 3. 地图分层

地图采用混合分层，而不是强制所有场景都套同一层级。

允许存在的层级包括：

- `大区域`
- `地点`
- `子区域`

当前约定：

- 系统允许 `大区域 -> 地点 -> 子区域`
- 但不是每一张地图都必须严格三层
- 小型场景可以直接使用 `地点 -> 子区域`
- 大型场景再引入 `大区域`

这样做的目的：

- 保持地图表达弹性
- 不让小地图被过度结构化
- 也不让大地图缺少组织层

## 4. 地图开放逻辑

地图开放采用混合逻辑。

当前约定：

- 主路径区域大多可达
- 关键区域逐步开放
- 高价值区域逐步开放
- 隐藏区域逐步开放

这意味着：

- 玩家不会一开局就被大量锁区困住
- 玩家也不会一开始就拿到整张地图的全部价值
- 探索推进感主要来自关键区域、隐藏区域和条件区域的逐步揭示

## 5. 场合与事件层

聚餐、集会、公开指认等“场合”采用混合方式承载。

当前约定：

- 它们依附于某个地点发生
- 但本质上按事件来管理

这意味着：

- 场合既受地图约束
- 也拥有自己的规则和状态
- 同一个地点在不同时间、不同事件下，可以承载完全不同的社会压力与风险结构

## 6. 当前结论

地图系统当前不追求把所有空间关系一次性做细，而是先确定三件事：

- 地图结构允许混合分层
- 地图开放采用“主路径可达，关键区域逐步开放”的节奏
- 场合作为依附地点的事件层存在

后续如果继续细化，优先讨论：

1. 哪些类型的区域应归入“主路径”
2. 哪些类型的区域应归入“关键 / 高价值 / 隐藏”
3. 事件层与 NPC 日程、公开指认、暴露度之间的关系
