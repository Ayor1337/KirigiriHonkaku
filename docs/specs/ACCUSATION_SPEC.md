# 指认与结局规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/specs/GAME_SPEC.md) 中拆出正式指认、伪胜、失败与结局相关设计。
> 当前只记录这一块的大框架，不展开流程级细节。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档当前只回答以下问题：

- 正式指认在这个游戏里是什么
- 公开场合为什么重要
- 游戏有哪些主要结果类型

## 2. 总体结论

当前指认与结局系统可以概括为：

**正式指认不是一个结算按钮，而是一个高风险、高影响、会改写局势的关键行动；玩家能否成功，不只取决于是否猜中真凶，还取决于场合、证据、局势、自保能力与行动方式。**

## 3. 正式指认的定位

当前约定：

- 玩家可以进行正式指认
- 玩家不止允许一次指认
- 但每次错误、过早或证据不足的指认，都会改变局势

这意味着：

- 指认是调查的高潮行为
- 指认本身也会暴露玩家
- 指认不是低成本试探按钮

## 4. 公开场合的重要性

当前约定：

- 正式指认应尽量在人多的公开场合进行
- 公开场合不是单纯人数问题，而是“是否形成公开情境”

公开场合至少可能来自：

- 系统预设事件
- 玩家主动召集
- NPC 行为自然形成
- 特殊剧情触发

而公开场合至少会带来以下效果：

- 更适合正式指认
- 降低被单独反杀的风险
- 旁观 NPC 可成为见证人
- 影响舆论和后续局势
- 放大成功或失败的结果

## 5. 私下指认的风险

当前约定：

- 如果玩家单独指认真凶，存在被当场反杀的风险
- 由于游戏带有 DND 气质，玩家也可能具备反制能力

这意味着：

- 指认既是推理动作，也是战术动作
- 场合错误时，即使怀疑正确，也可能付出极高代价

## 6. 成功条件

当前约定：

- 游戏不是只要猜中真凶就算胜利
- 猜中对象但证据不足，不算正常胜利
- 胜利判定采用混合式：关键硬证据 + 核心逻辑成立

这意味着：

- 指认不是竞猜
- 证据链和场合同样重要

## 7. 伪证与伪胜

当前约定：

- 玩家可以尝试做伪证
- 伪证的结果取决于属性与局势
- 玩家也可能通过暴力方式直接杀死真凶

因此系统允许存在非标准结果，例如：

- 伪证推动错误定案
- 通过暴力解决真凶而获得伪胜

## 8. 结果类型

当前确认的主要结果类型包括：

- 正常成功
- 证据不足失败
- 指认错误失败
- 被真凶当场反杀
- 玩家先下手击杀真凶，形成伪胜
- 玩家通过伪证推动错误定案
- 指认未结束案件，但强烈改变局势

## 9. 当前结论

指认与结局系统当前已经可以先按下面这句话理解：

这是一个把“定案”设计成高风险现场行为的方案，玩家不仅要判断谁是真凶，还要判断何时指认、在哪里指认、拿什么指认，以及自己是否承受得起这次行动带来的后果。

后续如果继续细化，优先讨论：

1. 正式指认的流程结构
2. 公开场合与私下场合的具体差异
3. 各类结局之间的正式判定边界
