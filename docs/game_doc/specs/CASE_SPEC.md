# 案件模板规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/specs/GAME_SPEC.md) 中拆出案件模板相关设计。
> 当前只记录案件设计的大方向，不展开到具体谜题和单案结构。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档当前只回答以下问题：

- 这类游戏的案件通常发生在什么样的舞台
- 案件中的人物关系通常以什么结构展开
- 案件整体更偏哪种本格推理风格

## 2. 总体结论

当前案件方向可以概括为：

**以经典封闭或半封闭空间为主舞台，以高密度人物关系和社会博弈为主结构，以时间差、不在场证明、伪装误导和关系隐瞒为主要谜题风格。**

## 3. 舞台方向

当前推荐的舞台优先级如下：

### 3.1 主方向：经典封闭空间

首选舞台应以经典封闭空间为主，例如：

- 宅邸
- 庄园
- 别馆
- 山庄

选择这类舞台的原因：

- 本格推理气质最稳
- 人物数量容易控制
- 地图边界清晰
- 方便设计隐藏区域、上锁空间和公开场合
- 更适合做“谁在什么时候去了哪里”的调查体验

### 3.2 次方向：半开放空间

第二优先级是边界明确、内部复杂的半开放舞台，例如：

- 学校
- 医院
- 修道院
- 剧院
- 旅馆

这类舞台的价值在于：

- 更容易带出职业差异和社会身份差异
- 更容易做多层地图与子区域
- 更容易自然产生公开场合和功能区差异

### 3.3 扩展方向：移动或特殊空间

后续可作为扩展舞台的方向包括：

- 邮轮
- 列车
- 孤岛营地

这类舞台的特点是：

- 氛围强
- 时间压力更自然
- 封闭性更容易成立
- 但前期不必作为主打方向

## 4. 人物关系方向

案件中的人物关系应保持高密度，而不是松散的陌生人集合。

当前推荐的人物关系结构包括：

- 家族 / 继承关系
- 主仆 / 屋内工作人员关系
- 同住者 / 共同行动者关系
- 同事 / 同学 / 同行关系
- 邀请者与宾客关系

整体方向上，案件更适合采用：

- 少量角色
- 明显关系链
- 多层次隐瞒
- 表面秩序下的潜在冲突

这样做的原因：

- 更利于玩家记忆角色
- 更利于设计证词矛盾和立场变化
- 更容易让 AI 驱动的对话产生张力

## 5. 谜题风格方向

案件整体不应只靠“找到一个关键物证”解决，而应更偏向复合型本格结构。

当前推荐的主风格包括：

- 时间差
- 不在场证明
- 伪装与误导
- 身份或关系隐瞒
- 空间结构利用

其中可持续反复使用的核心味道是：

- “谁在什么时候出现在什么地方”
- “谁在说谎，为什么说谎”
- “哪条证词和哪条事实对不上”
- “哪件事表面看合理，实际上是被布置出来的”

## 6. 案件体验方向

案件体验应更偏“社会博弈型推理”，而不是纯线索拼图。

这意味着：

- 玩家不只是找物证
- 玩家还要判断谁能信、谁在回避、谁在借机操纵局势
- 玩家问什么很重要，什么时候问、在什么场合问同样重要
- 调查越深入，世界对玩家的反作用越强

## 7. 当前结论

案件模板当前已经可以先按下面这句话理解：

这是一个以封闭舞台、小规模高关系密度角色群、以及“时间线 + 社会博弈 + 伪装误导”为核心的案件设计方向，主目标不是做成纯谜题盒，而是做成一个会反击玩家的推理现场。

后续如果继续细化，优先讨论：

1. 首批案件最适合先用哪一种舞台
2. 案件里的常见关系结构应如何复用
3. 哪几类谜题应作为项目的标志性风格
