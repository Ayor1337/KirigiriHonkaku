# Step 3 主循环落地阶段计划

> 本文档只覆盖项目落地的第三阶段：主循环落地阶段。
> 它用于承接 [OVERVIEW.md](./OVERVIEW.md) 与 [STEP2_WORLD_STATE_CORE.md](./STEP2_WORLD_STATE_CORE.md)，进一步明确阶段三的目标、范围、产物与完成标准。
> 当前仍然不拆到逐步任务清单。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 阶段定位

Step 3 的目标不是把整局案件全部做完，而是把最小可玩的调查主循环真正跑起来。

这一阶段只回答一个问题：

**怎样把已经稳定下来的世界状态核心，推进成一个“玩家动作可以进入、规则可以结算、世界会发生变化、下一步还能继续行动”的最小运行闭环。**

## 2. 阶段目标

本阶段的核心目标包括：

- 跑通最小调查主循环
- 固定 `Action -> Rule Resolution -> State Mutation -> Snapshot / Feedback` 的主执行链
- 让 `移动 / 调查 / 对话 / 时间推进 / NPC 基础调度` 形成连续可运行的动作闭环
- 让世界状态变化能够稳定反馈到后续动作输入
- 让后续 AI 叙事接入时，不需要反复回改主循环的硬规则骨架

一句话：

**先让世界真正动起来，而不是先让所有文本表现都到位。**

## 3. 阶段范围

Step 3 只覆盖主循环级落地，当前范围如下：

### 3.1 应落地的部分

- `Action` 的最小分类与进入引擎方式
- `move` 动作的路径与位置变更
- `investigate` 动作的基础结算
- `talk` 动作的基础会话触发
- 时间推进规则
- NPC 基础调度与位置更新
- 场景快照或场景反馈的最小生成机制
- `ActionResult` 对前端或调用方的最小可消费结构
- 主循环中的 Repository / UnitOfWork 协作方式

### 3.2 当前不纳入的部分

- AI 生成完整 NPC 台词
- 对话摘要和 `MEMORY.md` 更新
- 暴露度、公开场合、正式指认与结局闭环
- 复杂的 NPC 行为树或高级日程策略
- 侦探板的完整推理辅助能力
- 多案件模板和内容扩展能力

## 5. 阶段产物

Step 3 完成后，项目应至少形成以下稳定产物：

### 5.1 主循环执行链

- 玩家动作可以通过统一入口进入引擎
- 引擎可以根据动作类型进入对应规则分支
- 规则分支可以读写世界状态并提交结果
- `ActionResult` 能稳定返回动作后的结构化反馈

### 5.2 最小可玩的调查能力

- 玩家可以在地点之间移动
- 玩家可以对地点或对象执行调查
- 玩家可以与当前可对话 NPC 发起基础会话
- 时间会随着动作推进而变化
- NPC 会根据最小调度规则更新自身状态

### 5.3 场景反馈骨架

- 当前地点信息可以随着行动更新
- 可见 NPC、可调查对象、可触发对话等最小场景信息可以重新计算
- 调用方能够基于动作结果拿到下一步决策所需的最小上下文

## 6. 阶段完成标准

Step 3 是否完成，不看“AI 有没有表现力”，而看下面这些主循环标准是否成立：

- `move / investigate / talk` 至少具备一条完整执行路径
- 玩家动作确实会改变世界状态，而不是只返回静态文本
- 时间推进已经进入主循环，不再停留在概念层
- NPC 基础调度已经进入运行链路，而不是纯手工设定
- 一次动作产生的状态变化，能够影响下一次动作的可选项或结果
- 场景反馈已经能够反映位置、人物、线索可见性等基础变化
- 后续阶段可以在不推翻主循环硬规则骨架的前提下接入 AI 表达层

## 7. 阶段内的重点取舍

本阶段必须坚持以下取舍：

- 优先跑通动作闭环，不优先提升文本表现
- 优先固定执行链，不优先补复杂分支
- 优先保证状态推进真实发生，不优先追求结果丰富度
- 优先让规则层承担世界变更，不让 AI 越权结算硬状态

这意味着：

- 允许阶段三结束时“对话文本还比较朴素”
- 但不允许阶段三结束时“动作执行后世界没有真实变化”

## 8. 阶段结束后应能承接的下一步

Step 3 完成后，项目应自然进入以下下一层计划：

- AI 叙事接入
- 对话文本增强
- 记忆摘要与文本落盘

也就是：

- Step 3 负责“世界能动”
- Step 4 才开始负责“世界会说话”

## 9. 当前结论

Step 3 可以概括为：

**把这个项目从“世界状态已经能被稳定表示”推进到“玩家已经可以在规则约束下推动世界前进”的阶段，让移动、调查、对话、时间推进与 NPC 基础调度形成最小可玩的调查闭环，并为后续 AI 叙事接入提供稳定的主循环骨架。**
