# 真相模型规范草案

> 本文档用于从 [GAME_SPEC.md](C:/Users/ayor/PycharmProjects/KirigiriHonkaku/docs/GAME_SPEC.md) 中拆出“唯一真相”相关设计。
> 当前只记录真相系统的大框架，不展开字段级细节。
> 固定指令：Implementation Plan, Task List and Thought in Chinese

## 1. 文档定位

本文档当前只回答以下问题：

- 一局游戏中的“真相”是什么
- 真相和故事、NPC、线索之间是什么关系
- 真相模板应覆盖哪些层面

## 2. 总体结论

当前真相系统可以概括为：

**每一局在开始时先生成一个唯一真相，它是本局世界状态的事实源；故事、NPC 记忆、线索分布、误导与历史事件都围绕这套真相展开。**

## 3. 真相的定位

当前约定：

- 每一局开始时，系统先生成一个唯一真相
- 这套真相先于玩家调查存在
- 真相不随玩家推理而改变

这意味着：

- 真相是世界事实
- 玩家调查是在逼近真相，而不是共同创造真相
- 线索闭环、NPC 口径和案件复盘都需要回到这套事实源

## 4. 真相与其他系统的关系

真相不是单独存在的文本，而是整个案件结构的上游来源。

当前理解中：

- 开场故事围绕真相裁切信息
- NPC 的 `MEMORY.md` 由真相衍生出“知道什么、误会什么、隐瞒什么”
- 线索由真相决定其存在位置、价值和误导作用
- 世界历史与事件推进也围绕真相展开

## 5. 真相模板范围

当前真相模板至少覆盖以下部分：

- 案件结果
- 作案动机
- 作案手法
- 时间线
- 地点路径
- 证据分布
- 伪装与误导

当前不把以下内容直接纳入真相模板：

- 证词冲突点
- 玩家侧可逆推要点

## 6. 真相与推理的边界

当前约定：

- 真相负责描述“实际上发生了什么”
- 不负责描述“玩家应如何推出来”

这意味着：

- 真相是客观事实层
- 推理是玩家体验层
- 证词冲突、线索组织、指认链条属于从真相衍生出来的玩法结果，而不属于真相本体

## 7. 当前结论

真相系统当前已经可以先按下面这句话理解：

这是一个以“唯一真相先行”为前提的案件设计方案，真相负责给世界提供客观事实骨架，而玩家的调查、怀疑、误判和定案都围绕这套骨架展开。

后续如果继续细化，优先讨论：

1. 真相模板中“误导”应占多大比重
2. 真相和线索分布之间的映射规则
3. 真相和结案判定之间的衔接方式
